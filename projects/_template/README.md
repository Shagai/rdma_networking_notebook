# Project Template

## Goal

Describe the concrete thing the learner will build. Keep this focused on the
observable final behavior, not just the RDMA APIs being used.

## Concepts Practiced

- RDMA concept practiced by the project.
- Systems concept practiced by the project.
- Debugging or measurement skill practiced by the project.

## Prerequisites

- Learning-path document or lab that should be completed first.
- Tooling, hardware, or simulator requirement.
- Any assumed networking setup.

## Milestones

1. Build the smallest version that compiles and starts.
2. Establish connection setup and teardown.
3. Move one small payload correctly.
4. Add error handling and observability.
5. Measure or inspect the behavior.

## Verification

Document the commands that compile, run, and validate the project. Include the
expected output or observation for each command.

## Stretch Goals

- Optional improvement for performance.
- Optional improvement for reliability.
- Optional improvement for usability or diagnostics.
